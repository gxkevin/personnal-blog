<script lang="ts">
    import Vue from 'vue'
    import {Component,Prop} from 'vue-property-decorator';
    @Component //装饰器告诉typescript 这是一个vue组件，不加装饰器没办法处理下面的方法，数据声明
    export default class Home extends Vue{
        greeting = 'Hello'; //TS会自动猜测类型，知道为string类型。
      	@Prop(Number) xxx: number | undefined; //Prop装饰器告诉Vue.xx这是个prop.不是data
      	//undefined 解决xxx初始值的问题，但是后续使用xxx，必须检查是不是undefined，不然就报错。或者赋默认值
        //Number 告诉 Vue xxx是个number Vue不支持小写 运行时
      	//xxx 属性名字
      	// 告诉TS number|undefined 就是xxx的类型 编译时
        SayHello(greeting:string){ //函数的参数进行类型声明
            window.alert(greeting)
        };
        created(){
            this.SayHello(this.greeting);
          	console.log(this.xxx)
        }
    }
</script>

TypeScript好处:

1.代码自动提示更智能。

2.声明了类型，代码写错了，会提前告诉我。编译报错，无法变成JS，严谨。

3.检查调用的方法到底能不能调用，调用错了会报错。
